# Security Policy

## Reporting a Vulnerability

The Stihia team takes security seriously — especially given that this SDK is part of an AI security platform.

If you discover a security vulnerability in this project, please report it responsibly:

1. **Do NOT open a public issue.**
2. Use [GitHub's private vulnerability reporting](https://github.com/stihia-ai/stihia-sdk-python/security/advisories/new) to submit your report.
3. Include as much detail as possible: steps to reproduce, impact assessment, and any suggested fixes.

## Response Timeline

- **Acknowledgment**: Within 48 hours of receiving the report.
- **Assessment**: Within 5 business days, we will confirm the vulnerability and determine its impact.
- **Fix**: We aim to release a patch within 30 days of confirmation, depending on severity and complexity.

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| Latest  | :white_check_mark: |

## Scope

This policy covers the `stihia` Python SDK package. For issues related to the Stihia API service itself, please contact the Stihia team directly by sending an email to [support@stihia.ai](mailto:support@stihia.ai).
